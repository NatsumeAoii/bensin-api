# Bensin-API — Full Project Improvement Audit

Generated from a complete read of every source file, config, component, page,
utility, store, Python pipeline module, schema, test, workflow, generated data
sample, and documentation file in the repository. Every suggestion below is
grounded in code that was actually read.

## Implementation Status

All categories except **5 (Feature Expansion and Enhancement)** have been
implemented. Verified green after the work:

- Frontend: `npm run lint`, `npm run typecheck`, `npm run format:check`,
  `npm run build` (JS < 200 KB, CSS < 75 KB gzipped), and `vitest run`
  (38 files, 213 tests) all pass.
- Pipeline: `python -m pytest pipeline/tests/` passes (49 tests);
  `python -m pipeline.sanity_check` passes against committed `v1/`.

Notable resolutions and deviations from the original suggestions:

- **#4** — No regeneration needed: the committed `v1/` product names were
  already canonical (`BIOSOLAR`, `BIOSOLAR NON SUBSIDI`). The fix was correcting
  the docs that wrongly claimed drift, not changing data.
- **#16/#21** — The font-load and CSP work was made CSP-safe: the strict CSP is
  injected only into the production build via a Vite plugin (not the dev
  server, where it would break HMR), and the font uses `preload` + `display=swap`
  instead of an inline `onload` handler the CSP would block.
- **#38** — PWA icons: declared the SVG for both `any` and `maskable` purposes.
  Real 192/512 PNGs still require a binary asset and are left for a follow-up.
- **#65** — Coverage thresholds set as a regression floor (60% lines/stmts,
  48% branches/functions) reflecting the untested page components, rather than
  an aspirational 70%.
- Suggestions in **Category 5** (#41–#48) were intentionally skipped per request.
- A few items remain as follow-ups (binary PWA icons, full i18n extraction
  #89/#90, roving-tabindex #33, interior chart ticks #32) — noted but not done.

The original audit follows unchanged below.

## Phase 1 — Project Map

**Stack:** React 19 + TypeScript (strict) + Vite 8 (Rolldown) + Tailwind CSS 4 +
Zustand 5 + React Router 7, with a Python 3.12 data pipeline (Pydantic +
Hypothesis). LightningCSS for CSS transform/minify. Lucide React for icons.

**Shape:** Two decoupled halves.

1. **Python pipeline** (`pipeline/`) reads `price.json`, normalizes prices and
   slugs, validates with Pydantic, writes static JSON to `v1/` (index, nasional,
   per-province) plus append-only change-based history to `v1/history/`.
2. **React SPA** (`src/`) fetches those static JSON files from a hard-coded
   GitHub Pages base URL and renders a fuel-price dashboard.

**Routes:** `/` (province list), `/provinsi/:slug` (detail + history chart),
`/nasional` (national comparison), `*` (404).

**State:** `fuel-store` (Zustand) caches index/province/national with per-key
retry tracking; `theme-store` manages light/dark.

**Utilities:** `date`, `format`, `history-range`, `products`, `search`, `slug`,
`sort`, `use-document-title`, `use-visibility-refresh`.

**Tests:** 19 unit + 14 fast-check property tests (frontend); 5 pytest modules
incl. Hypothesis (pipeline).

**CI/CD:** `sync.yml` (scheduled 6h data sync, direct push to main),
`deploy-pages.yml` (build + deploy to GitHub Pages).

**Notable observed issues feeding the suggestions below:**

- A stray `web/node_modules/` directory with no `web/` source (orphan).
- Documentation (README, ARCHITECTURE, SECURITY) describes a PR-based sync
  workflow with `create-pull-request` + `gh pr merge`, but `sync.yml` actually
  does a direct `git push origin HEAD:main`. README also says sync runs
  "hourly" while the cron is every 6 hours.
- Committed `v1/` snapshots have product-name drift vs `PRODUCT_CANONICAL_MAP`.
- No frontend lint/typecheck/test gate in CI (only the build runs in deploy).
